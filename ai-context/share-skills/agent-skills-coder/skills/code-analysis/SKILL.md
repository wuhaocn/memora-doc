# code-analysis

## 描述
分析代码库架构并生成结构化的架构文档，包括组件关系、依赖图和工作流等内容。

## 使用场景
当需要了解代码库结构、组件划分、依赖关系、工作流程时，使用此技能可以生成清晰的架构文档。

## 指令

1. **明确分析目标**：确定需要分析的代码范围和分析目标（如整体架构、特定模块、组件关系等）

2. **收集代码信息**：
   - 识别主要模块和组件
   - 分析目录结构和文件组织
   - 收集关键类和接口定义
   - 识别依赖关系

3. **分析组件关系**：
   - 绘制组件之间的依赖关系图
   - 识别模块间的调用关系
   - 分析数据流向

4. **分析工作流**：
   - 识别关键业务流程
   - 分析请求处理流程
   - 识别状态转换

5. **生成架构文档**：
   - 使用 Markdown 格式
   - 包含目录结构说明
   - 包含组件关系图（使用 Mermaid 或文本描述）
   - 包含关键接口和类说明
   - 包含数据流向说明

## 输出格式

架构文档应包含以下部分：

1. **概述**：代码库的整体描述和主要功能
2. **目录结构**：主要目录和文件的组织方式
3. **模块划分**：主要模块及其职责
4. **组件关系**：组件间的依赖和调用关系（可用 Mermaid 图表示）
5. **关键接口**：重要的接口和类定义
6. **数据流向**：数据在系统中的流转过程
7. **工作流程**：关键业务流程的说明

## 示例

### 输入示例
```
分析项目的用户管理模块架构，包括 Controller、Service、DAO 层的结构和关系
```

### 输出示例
生成包含模块结构、类关系图、接口定义和调用流程的架构文档
