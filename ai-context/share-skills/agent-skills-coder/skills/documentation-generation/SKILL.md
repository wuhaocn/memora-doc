## 使用场景
当需要生成API文档、技术设计文档、用户手册等文档时，使用此技能可以确保生成的文档符合项目的文档规范和最佳实践。

## 指令
1. **识别文档类型**：确定需要生成的文档类型
   - API文档（API Guide）
   - 技术设计文档（Technical Design Document）
   - 开发指南（Development Guide）
   - 用户手册（User Manual）
   - 部署文档（Deployment Guide）
   - 监控文档（Monitoring Guide）

2. **生成文档结构**：
   - 根据文档类型生成相应的文档结构
   - 包含必要的章节和内容
   - 遵循项目的文档规范

3. **使用Markdown格式**：
   - 使用Markdown语法编写文档
   - 遵循统一的Markdown样式
   - 使用适当的标题层级
   - 包含代码块、表格、列表等元素

4. **文档内容完整性**：
   - 包含文档的所有必要章节
   - 提供清晰的文档说明
   - 包含示例代码和使用示例
   - 遵循项目的文档模板

5. **API文档生成**：
   - 从代码中提取API信息
   - 生成API端点、请求/响应格式
   - 包含错误码和示例
   - 遵循项目的API文档规范

6. **技术文档生成**：
   - 包含系统架构、组件设计
   - 描述数据模型和流程
   - 提供部署和配置说明
   - 包含监控和维护信息

## 参数
- `doc_type`: 文档类型，可选值为 `api`、`technical`、`development`、`user`、`deployment`、`monitoring`，必填
- `module_name`: 模块名称，如 `template`、`proxychat`，必填
- `api_type`: API类型，可选值为 `internal`（内部API）、`external`（外部API），仅当doc_type为api时必填
- `version`: 文档版本，如 `1.0.0`，可选，默认值为 `1.0.0`
- `base_path`: API基础路径，如 `/v3/internal/agent/template`，仅当doc_type为api时必填
- `endpoints`: API端点列表，格式为 `[{"path": "/create.json", "method": "POST", "description": "创建模板"}]`，仅当doc_type为api时必填
- `include_examples`: 是否包含示例代码，布尔值，可选，默认值为 `true`
- `include_schema`: 是否包含数据模型，布尔值，可选，默认值为 `true`

## 示例

### 输入示例
```
调用 documentation-generation skills，生成一个内部API文档，模块名称为 template，API基础路径为 /v3/internal/agent/template，API端点包括 /create.json（POST，创建模板）、/get.json（POST，获取模板）、/update.json（POST，更新模板）、/delete.json（POST，删除模板），包含示例代码和数据模型
```

```markdown
# 内部API开发指南 - Template模块

## 1. 概述

Template模块提供了模板管理的内部API，用于系统内组件之间的模板操作。

## 2. API设计原则

### 2.1 路径约定
- 基础路径: `/v3/internal/agent/template`

### 2.2 HTTP方法
- 所有内部API使用**POST**方法

### 2.3 请求/响应格式
- 内容类型: `application/json`
- 响应策略: NON_NULL（排除响应中的null字段）

### 2.4 认证
- 内部API需要扩展认证
- 使用系统级令牌进行安全认证

## 3. API端点

### 3.1 创建模板

*   **URL**: `/v3/internal/agent/template/create.json`
*   **Method**: `POST`
*   **Description**: 创建新的模板

**请求参数 (Body)**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| **templateId** | String | 是 | 模板ID，唯一标识 |
| **name** | String | 是 | 模板名称 |
| **type** | String | 是 | 模板类型，如 `prompt` |
| **template** | Object | 是 | 模板内容，包含 `promptText` 和 `arguments` |

**请求示例**

```json
{
  "templateId": "pt_public_summary_v1",
  "name": "公共摘要模板",
  "type": "prompt",
  "template": {
    "promptText": "请总结以下文本：{text}",
    "arguments": [
      {
        "name": "text",
        "description": "需要摘要的文本",
        "required": true
      }
    ]
  }
}
```

**响应示例**

```json
{
  "code": 200,
  "message": "success"
}
```

### 3.2 获取模板

*   **URL**: `/v3/internal/agent/template/get.json`
*   **Method**: `POST`
*   **Description**: 获取指定模板

**请求参数 (Body)**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| **templateId** | String | 是 | 模板ID |

**请求示例**

```json
{
  "templateId": "pt_public_summary_v1"
}
```

**响应示例**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "templateId": "pt_public_summary_v1",
    "name": "公共摘要模板",
    "type": "prompt",
    "template": {
      "promptText": "请总结以下文本：{text}",
      "arguments": [
        {
          "name": "text",
          "description": "需要摘要的文本",
          "required": true
        }
      ]
    },
    "createdAt": 1677652288000,
    "updatedAt": 1677652288000
  }
}

# Template模块技术设计文档

## 1. 概述

Template模块用于管理系统中的各种模板，包括提示模板、响应模板等。本文档描述了Template模块的技术设计，包括系统架构、组件设计、数据模型等。

## 2. 系统架构

### 2.1 模块结构

```
com.rcloud.template/            # 根包
├── internal.http/             # 内部HTTP接口层
├── service/                   # 业务逻辑层
├── data/                      # 数据访问层
└── modedefine/                 # 定义层
```

### 2.2 核心组件

| 组件 | 职责 | 文件位置 |
| :--- | :--- | :--- |
| TemplateController | 处理HTTP请求 | com.rcloud.template.internal.http.TemplateController |
| TemplateService | 实现业务逻辑 | com.rcloud.template.service.TemplateService |
| TemplateDataImpl | 数据访问实现 | com.rcloud.template.data.impl.TemplateDataImpl |
| TemplateConfig | 配置类 | com.rcloud.template.modedefine.config.TemplateConfig |

## 3. 数据模型

### 3.1 数据库表设计

```sql
CREATE TABLE `CC_Template` (
  `id` bigint(20) unsigned NOT NULL AUTO_INCREMENT,
  `templateId` varchar(64) NOT NULL COMMENT '模板ID',
  `name` varchar(255) NOT NULL COMMENT '模板名称',
  `type` varchar(32) NOT NULL COMMENT '模板类型',
  `template` json NOT NULL COMMENT '模板内容',
  `createdAt` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updatedAt` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_templateId` (`templateId`)
  
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='模板表';
```
