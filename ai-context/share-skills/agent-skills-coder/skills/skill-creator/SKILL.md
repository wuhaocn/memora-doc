# skill-creator

## 描述
用于创建新的共享技能或工具兼容技能。该技能可以帮助开发者快速创建符合规范的技能，确保技能的一致性和可复用性。

## 使用场景
当需要创建新的技能时，使用此技能可以确保技能符合项目规范和最佳实践。

## 指令

1. **确定技能类型**：确定需要创建的技能类型，如 `code-analysis`, `api-design`, `frontend-design` 等

2. **设计技能内容**：设计技能的名称、描述、使用场景、指令、参数和示例等

3. **创建技能文件**：创建技能的基本文件结构和内容

4. **验证技能**：验证技能是否符合规范和要求

## 技能目录结构

```
ai-context/share-skills/{collection}/skills/{skill-name}/
└── SKILL.md    # 技能描述文件
```

## SKILL.md 格式

技能文件应包含以下部分：

```markdown
---
name: skill-name
description: 技能的简短描述
---

# skill-name

## 描述
技能的详细描述，说明技能的目的和使用场景。

## 使用场景
何时使用此技能。

## 指令
技能的执行步骤，按顺序列出。

## 参数
技能需要的参数（如适用）。

## 示例
输入和输出示例。
```

## 技能命名规范

- 使用 kebab-case 格式（如 `frontend-design`）
- 名称应清晰、描述性
- 避免使用缩写，除非被广泛理解

## 技能内容要求

1. **描述清晰**：技能描述应准确说明技能的功能和用途
2. **指令明确**：指令应具体、可执行，按步骤列出
3. **示例完整**：提供输入和输出示例，帮助理解技能的使用方式
4. **符合规范**：遵循项目的代码规范和最佳实践

## 参数

- `skill_name`: 技能名称，使用 kebab-case 格式
- `skill_type`: 技能类型，如 `code-analysis`, `api-design`, `frontend-design` 等
- `description`: 技能的简短描述

## 示例

### 输入示例
```
调用 skill-creator 技能，创建一个名为 "data-visualization" 的技能，类型为 "frontend-design"，描述为 "创建数据可视化组件和图表"
```

### 输出示例
生成符合规范的技能文件结构，包括完整的 SKILL.md 文件

## 使用建议

1. **技能命名**：使用清晰、描述性的技能名称
2. **技能类型**：根据技能的功能选择合适的技能类型
3. **技能内容**：确保技能的内容完整、清晰、准确
4. **技能测试**：创建后验证技能是否符合规范和要求

## 后续步骤

1. 完善技能的内容和示例
2. 测试技能是否符合规范和要求
3. 根据使用反馈持续优化技能
