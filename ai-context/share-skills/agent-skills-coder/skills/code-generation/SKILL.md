## 使用场景
当需要生成控制器、服务、模型等核心组件的代码时，使用此技能可以确保生成的代码符合项目规范和最佳实践。

## 指令
1. **识别代码类型**：确定需要生成的代码类型
   - 控制器（Controller）
   - 服务（Service）
   - 数据访问层（DAO/Repository）
   - 模型类（Entity/Model）
   - 请求/响应模型（DTO）
   - 配置类
   - 异常处理

2. **生成代码结构**：
   - 根据代码类型生成相应的类结构
   - 包含必要的注解和接口
   - 实现基础的方法签名

3. **遵循命名约定**：
   - 类名使用PascalCase
   - 方法名和变量名使用camelCase
   - 包名使用小写字母和点分隔

4. **实现日志记录**：
   - 使用SLF4J进行日志记录
   - 在关键方法中添加日志

5. **实现异常处理**：
   - 使用标准化错误码
   - 包含清晰的错误信息
   - 抛出适当的异常类型

6. **实现参数验证**：
   - 为请求模型实现Validatable接口
   - 使用PropertyValidator进行参数校验
   - 验证必填字段、数据类型和长度

7. **生成示例代码**：
   - 提供完整的示例代码
   - 包含核心功能的实现
   - 遵循项目的代码规范

## 参数
- `code_type`: 代码类型，可选值为 `controller`、`service`、`dao`、`model`、`dto`、`config`、`exception`，必填
- `class_name`: 类名，如 `TemplateController`，必填
- `package_name`: 包名，如 `com.rcloud.template.internal.http`，必填
- `base_class`: 基类，如 `BaseController`，可选
- `interfaces`: 实现的接口列表，格式为 `["Interface1", "Interface2"]`，可选
- `fields`: 字段列表，格式为 `[{"name": "field1", "type": "String", "access": "private", "required": true}]`，可选
- `methods`: 方法列表，格式为 `[{"name": "method1", "returnType": "String", "parameters": [{"name": "param1", "type": "String"}], "access": "public"}]`，可选
- `api_type`: API类型，可选值为 `internal`（内部API）、`external`（外部API），仅当code_type为controller时必填
- `module_name`: 模块名称，如 `template`、`proxychat`，仅当code_type为controller时必填

package com.rcloud.template.internal.http.model;

import com.fasterxml.jackson.annotation.JsonProperty;
import com.rcloud.http.flux.starter.exception.UnifyException;
import com.rcloud.http.flux.starter.model.PropertyValidator;

/**
 * 创建模板请求
 */
public class CreateTemplateRequest implements Validatable {

    @JsonProperty("templateId")
    private String templateId;

    @JsonProperty("name")
    private String name;

    @JsonProperty("type")
    private String type;

    @JsonProperty("template")
    private Template template;

    // Getters and setters

    @Override
    public void validate() throws UnifyException {
        PropertyValidator.of(templateId, "templateId", null)
                .required()
                .isString()
                .notBlank()
                .maxLength(64);

        PropertyValidator.of(name, "name", null)
                .required()
                .isString()
                .notBlank()
                .maxLength(255);

        PropertyValidator.of(type, "type", null)
                .required()
                .isString()
                .notBlank()
                .maxLength(32);

        PropertyValidator.of(template, "template", null)
                .required()
                .isObject();
    }
}
