# 通过接口获取 Skills 列表并更新文档

本文档说明如何通过 GitHub API 接口获取 Agent Skills 列表并更新到本地文档目录。

## 实现方案

我们使用 Python 脚本通过 GitHub API 获取 `libukai/awesome-agent-skills` 仓库的内容，解析其中的技能列表，然后生成更新后的 Markdown 文档。

## 工具准备

1. **Python 环境**：需要 Python 3.6+ 环境
2. **依赖库**：
   - `requests`：用于发送 HTTP 请求
   - `beautifulsoup4`：用于解析 HTML（可选，根据实际需要）

## 安装依赖

```bash
pip install requests beautifulsoup4
```

## 使用方法

### 1. 运行更新脚本

```bash
# 在 scripts 目录下运行
python /Users/wuhao/data/ai/doc-studio/doc/业界Skills/update_skills.py
```

### 2. 脚本功能说明

- **获取内容**：通过 GitHub API 获取 `libukai/awesome-agent-skills` 仓库的 README.md 文件
- **解析技能**：从 README.md 中解析技能列表
- **生成文档**：将解析出的技能列表与原有技能整合，生成新的 Markdown 文档
- **更新文件**：将生成的文档保存到指定目录

### 3. 配置选项

脚本中的配置项可以根据需要修改：

```python
# 配置
GITHUB_REPO = "libukai/awesome-agent-skills"  # GitHub 仓库
GITHUB_API_URL = f"https://api.github.com/repos/{GITHUB_REPO}/contents/README.md"  # API URL

```

## 定时更新

可以设置定时任务，定期运行脚本更新技能列表：

### 使用 crontab（Linux/macOS）

```bash
# 编辑 crontab
crontab -e

# 添加定时任务（每天凌晨 1 点更新）
0 1 * * * python /Users/wuhao/data/ai/doc-studio/doc/业界Skills/update_skills.py
```

### 使用 Task Scheduler（Windows）

1. 打开任务计划程序
2. 创建新任务
3. 设置触发器（如每天凌晨 1 点）
4. 设置操作为运行 Python 脚本
5. 保存任务

## 注意事项

1. **GitHub API 速率限制**：未认证的 API 请求有速率限制，建议使用 GitHub Token 进行认证
2. **网络连接**：需要网络连接才能获取 GitHub 仓库内容
3. **解析逻辑**：解析逻辑基于 README.md 的特定格式，若仓库格式变更，可能需要调整解析代码
4. **文件权限**：确保脚本有写入输出目录的权限

## 故障排除

- **API 请求失败**：检查网络连接和 GitHub API 速率限制
- **解析失败**：检查 README.md 格式是否变更，调整解析逻辑
- **写入失败**：检查文件权限和目录是否存在

## 扩展功能

可以根据需要扩展脚本功能：

1. **支持多个数据源**：从多个仓库或 API 获取技能列表
2. **分类整理**：根据技能类型进行分类整理
3. **自动测试**：验证技能链接的有效性
4. **通知机制**：当有新技能添加时发送通知

## 示例输出

运行脚本后，会在指定目录生成更新后的 `业界优秀Skills.md` 文件，包含从 GitHub 获取的技能列表和原有技能。

通过这种方式，可以定期从 GitHub 上的权威资源获取最新的 Agent Skills 列表，保持本地文档的时效性和完整性。
