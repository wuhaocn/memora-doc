# DocStudio 资源库设计文档

## 1. 概述

### 1.1 功能简介
资源库是 DocStudio 系统的一个重要功能模块，用于集中管理和组织各类素材、文案和提示词等内容。用户可以通过资源库快速访问、管理和复用这些资源，提高创作效率和内容质量。

### 1.2 核心概念
- **资源（Resource）**: 资源库中的基本单元，包括素材、文案、提示词等类型
- **标签（Tag）**: 用于对资源进行分类和标记的关键词
- **资源类型（ResourceType）**: 资源的分类，如素材、文案、提示词等

### 1.3 设计目标
- 提供统一的资源管理界面
- 支持多种类型资源的管理
- 实现灵活的标签分类系统
- 支持资源的搜索和筛选
- 支持资源与文档的关联
- 提供良好的用户体验

## 2. 数据库设计

### 2.1 资源表（resource）
```sql
CREATE TABLE `resource` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `name` VARCHAR(200) NOT NULL COMMENT '资源名称',
  `description` VARCHAR(500) COMMENT '资源描述',
  `type` VARCHAR(50) NOT NULL COMMENT '资源类型: material, copywriting, prompt',
  `content` LONGTEXT COMMENT '资源内容',
  `content_url` VARCHAR(500) COMMENT '资源文件URL（用于素材类型）',
  `content_type` VARCHAR(100) COMMENT '内容类型（MIME类型）',
  `user_id` BIGINT NOT NULL COMMENT '创建者ID',
  `status` TINYINT DEFAULT 1 COMMENT '1:正常 0:删除',
  `is_public` TINYINT DEFAULT 0 COMMENT '是否公开',
  `view_count` INT DEFAULT 0 COMMENT '查看次数',
  `download_count` INT DEFAULT 0 COMMENT '下载次数',
  `sort_order` INT DEFAULT 0 COMMENT '排序顺序',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_type` (`type`),
  INDEX `idx_status` (`status`),
  INDEX `idx_created_at` (`created_at`)
);
```

### 2.2 标签表（tag）
```sql
CREATE TABLE `tag` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `name` VARCHAR(100) NOT NULL COMMENT '标签名称',
  `description` VARCHAR(255) COMMENT '标签描述',
  `user_id` BIGINT NOT NULL COMMENT '创建者ID',
  `status` TINYINT DEFAULT 1 COMMENT '1:正常 0:删除',
  `resource_count` INT DEFAULT 0 COMMENT '关联资源数量',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_name` (`name`),
  INDEX `idx_status` (`status`)
);
```

### 2.3 资源标签关联表（resource_tag）
```sql
CREATE TABLE `resource_tag` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `resource_id` BIGINT NOT NULL COMMENT '资源ID',
  `tag_id` BIGINT NOT NULL COMMENT '标签ID',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_resource_tag` (`resource_id`, `tag_id`),
  INDEX `idx_resource_id` (`resource_id`),
  INDEX `idx_tag_id` (`tag_id`),
  FOREIGN KEY (`resource_id`) REFERENCES `resource`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`tag_id`) REFERENCES `tag`(`id`) ON DELETE CASCADE
);
```

### 2.4 资源使用记录表（resource_usage）
```sql
CREATE TABLE `resource_usage` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `resource_id` BIGINT NOT NULL COMMENT '资源ID',
  `document_id` BIGINT COMMENT '使用资源的文档ID',
  `knowledge_base_id` BIGINT COMMENT '使用资源的知识库ID',
  `user_id` BIGINT NOT NULL COMMENT '使用资源的用户ID',
  `usage_type` VARCHAR(50) COMMENT '使用类型: insert, reference',
  `usage_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '使用时间',
  INDEX `idx_resource_id` (`resource_id`),
  INDEX `idx_document_id` (`document_id`),
  INDEX `idx_user_id` (`user_id`),
  FOREIGN KEY (`resource_id`) REFERENCES `resource`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`document_id`) REFERENCES `document`(`id`) ON DELETE SET NULL,
  FOREIGN KEY (`knowledge_base_id`) REFERENCES `knowledge_base`(`id`) ON DELETE SET NULL
);
```

#### 4.5.2 tagSlice
```javascript
const tagSlice = createSlice({
  name: 'tag',
  initialState: {
    tags: [],
    loading: false,
    error: null
  },
  reducers: {
    setTags: (state, action) => {
      state.tags = action.payload;
    },
    setLoading: (state, action) => {
      state.loading = action.payload;
    },
    setError: (state, action) => {
      state.error = action.payload;
    },
    addTag: (state, action) => {
      state.tags.push(action.payload);
    },
    updateTag: (state, action) => {
      const index = state.tags.findIndex(tag => tag.id === action.payload.id);
      if (index !== -1) {
        state.tags[index] = action.payload;
      }
    },
    removeTag: (state, action) => {
      state.tags = state.tags.filter(tag => tag.id !== action.payload);
    }
  }
});
```

## 5. 功能实现

### 5.1 资源类型实现

#### 5.1.1 素材类型
- **内容格式**: 文件URL或Base64编码
- **支持的文件类型**:
  - 图片: JPG, PNG, GIF, SVG
  - 视频: MP4, WebM
  - 音频: MP3, WAV
  - 其他: PDF, ZIP
- **预览方式**: 根据文件类型显示相应的预览

#### 5.1.2 文案类型
- **内容格式**: 纯文本或富文本
- **支持的格式**:
  - 标题文案
  - 描述文案
  - 宣传文案
  - 邮件模板
- **预览方式**: 文本预览，支持富文本格式

#### 5.1.3 提示词类型
- **内容格式**: 文本
- **支持的类型**:
  - 通用提示词
  - 特定模型提示词
  - 场景提示词
- **预览方式**: 文本预览，支持语法高亮

### 5.2 标签系统实现

#### 5.2.1 标签创建和管理
- 支持用户自定义标签
- 提供常用标签建议
- 标签名称唯一性检查
- 标签描述和颜色设置（可选）

#### 5.2.2 标签关联
- 支持为资源添加多个标签
- 支持通过标签搜索和筛选资源
- 支持批量标签操作
- 标签使用统计和热门标签展示

### 5.3 资源搜索和筛选

#### 5.3.1 搜索功能
- 支持关键词搜索资源名称和描述
- 支持按资源类型搜索
- 支持按标签搜索
- 支持组合搜索条件

#### 5.3.2 筛选功能
