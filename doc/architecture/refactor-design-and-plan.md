# 在线文档系统重构设计与计划

## 1. 文档定位

本文档是 Memora 当前阶段的路线与实施计划文档。

它是以下事实的唯一来源：

- 阶段评估
- 里程碑
- 实施顺序
- 风险与依赖

它不重复完整产品基线，也不重复完整技术基线。

---

## 2. 当前阶段判断

### 2.1 当前系统状态

仓库已经从之前相对分散的方向收回到在线文档主线。

现在可以明确说：

- 主链路已经存在
- 可演示闭环已经存在
- 文档体系已经完成一次收敛

仍不能明确说：

- 真实认证已经达到生产可用
- 系统已经适合正式生产上线
- Web 端产品体验已经完成收口

### 2.2 当前核心结论

正确策略不是继续扩模块，而是继续收紧这条主链路：

`tenant -> knowledge base -> document tree -> reading/editing -> versions -> permissions`

---

## 3. 里程碑

| 里程碑 | 目标 | 完成定义 |
| --- | --- | --- |
| M0 | 方向统一 | 文档、模块职责、主流程口径统一 |
| M1 | 基础平台可运行 | 登录、工作台、知识库、文档树、版本、最小权限可运行 |
| M2 | 内容管理闭环 | 知识库管理、文档组织、权限交互稳定 |
| M3 | Web 产品体验闭环 | 搜索、阅读体验、版本体验、权限交互进一步收敛 |
| M4 | 基础生产能力 | 真实认证、受控阅读/发布、审计达到基础可用 |

当前判断：

- `M0` 已完成
- `M1` 基本完成
- `M2` 进行中
- `M3/M4` 未完成

---

## 4. 当前优先级

### 4.1 第一优先级

- 真实认证与真实租户 / 用户上下文
- 更清晰的权限错误语义与前端反馈
- 主链路稳定验证
- README 与 canonical 文档持续一致

### 4.2 第二优先级

- 更完整的知识库权限交互
- 搜索
- 更好的版本 diff 体验
- 归档 / 回收站能力

### 4.3 第三优先级

- 受控阅读
- 发布与发布回滚
- 审计

### 4.4 第四优先级

- 外围接入能力边界确认

---

## 5. 阶段计划

### 5.1 P0 稳定化

目标：

- 从“可演示”提升到“适合持续内部联调”

重点：

- 恢复后端本地测试 / 构建环境
- 固化 backend / web 验证命令
- 强化权限与主链路测试
- 收紧异常处理与状态反馈

退出标准：

- Web lint/build 稳定
- Backend 测试执行可恢复
- 主流程冒烟清单无结构性阻塞

### 5.2 P1 产品化

目标：

- 把当前文档工作区推进到可用的内部知识产品

重点：

- 搜索
- 更强的权限交互
- 更好的版本行为
- 归档 / 回收站

退出标准：

- 用户能稳定搜索知识
- 管理员能清晰配置访问边界
- 版本与恢复可理解、可使用

### 5.3 P2 受控阅读与发布

目标：

- 把内部阅读、受控阅读、公开发布正式拆成独立产品流

重点：

- 受控阅读入口
- 发布状态
- 发布树 / 发布面
- 审核与撤销流程

退出标准：

- 阅读与发布语义清晰分离
- 访问边界可验证
- 已发布内容可回滚

### 5.4 P3 外围能力评估

目标：

- 在 Web 主产品稳定后，再决定是否扩展其他外围能力

重点：

- 外围能力是否仍有业务必要性
- 新能力边界如何与 Web 主链路解耦
- 是否需要独立仓库或独立产品线承载

退出标准：

- 外围能力扩展方向有明确结论
- 外围能力不再影响当前 Web 主线判断

---

## 6. 工作流拆分

### 6.1 Backend

- 认证与会话替换
- 权限与审计
- 搜索接口
- 发布接口

### 6.2 Web

- 搜索入口与结果页
- 权限配置交互
- 阅读 / 发布状态反馈
- 更好的版本 diff
- 统一异常与空状态处理
- 页面信息架构与操作流收口

### 6.3 Documentation

- 只维护 canonical 文档
- 行业参考与基线文档分开
- 不再引入总结类平行文档

---

## 7. 风险与依赖

### 7.1 主要风险

- 没有真实认证，权限就无法进入生产级
- 没有搜索，知识库产品价值会明显受限
- 如果文档再次分裂，产品与工程认知会再次漂移
- 如果过早重启外围能力，Web 主链路会再次失焦

### 7.2 主要依赖

- Backend Gradle 环境恢复
- Node 工具链稳定
- 前后端错误语义对齐

---

## 8. 当前阶段出口标准

当前直接目标应当是 `M2`。

当以下条件满足时，可视为 `M2` 完成：

- 知识库管理闭环稳定
- 文档树主流程稳定
- 权限配置交互可用
- canonical 文档持续与代码行为保持一致
- Web 端主操作流已经清晰、连续、可理解
