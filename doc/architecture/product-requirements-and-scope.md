# 在线文档系统产品需求与范围

## 1. 文档定位

本文档是 Memora 当前阶段的产品基线文档。

它是以下事实的唯一来源：

- 当前产品定位
- 当前产品设计
- 目标产品设计
- 范围与功能边界
- 角色、场景与成功标准

---

## 2. 产品定位

Memora 当前定位为：

`面向企业知识管理的在线文档系统`

当前主产品路径聚焦：

- 多租户工作区隔离
- 知识库管理
- 文档树管理
- 阅读 / 编辑双模式
- 版本历史
- 受控阅读与权限边界

当前不定位为：

- 通用 CMS
- 大而全协作办公套件
- AI 优先产品中心
- 资源库优先平台
- 通用块编辑器平台
- 独立桌面产品线

---

## 3. 产品目标

### 3.1 当前阶段目标

当前阶段只服务一条主链路：

`login -> workspace -> knowledge base -> document tree -> reading/editing -> versions -> permissions`

目标不是铺广，而是把这条链路做得稳定、易理解、可扩展。

### 3.2 当前目标形态

Memora 当前目标产品形态应是：

`企业知识库 + 文档工作区`

当前阶段重点应具备：

- 企业知识治理能力
- 清晰的信息结构
- 稳定的阅读与编辑体验
- 可靠的版本与恢复机制
- 明确的权限与受控阅读边界

---

## 4. 用户角色

### 4.1 租户管理员

关注：

- 管理工作区
- 管理知识库
- 管理成员与权限
- 管理受控阅读边界

### 4.2 知识库负责人

关注：

- 维护知识结构
- 管理目录与文档
- 保证文档质量
- 管理知识库级成员与权限

### 4.3 编辑者

关注：

- 创建与维护文档
- 调整目录结构
- 生成与查看版本

### 4.4 阅读者

关注：

- 快速找到内容
- 阅读有权限的文档
- 查看最新有效版本
- 通过受控阅读入口直达内容

---

## 5. 当前产品设计

### 5.1 当前产品闭环

当前活跃闭环：

`login -> workspace -> knowledge base -> document tree -> reading/editing -> versions -> permissions`

闭环要求：

- 用户能从工作台进入任意有权限的知识库
- 用户能在知识库内创建、整理、阅读、编辑文档
- 每次正文更新都生成版本记录
- 阅读链接只是直达入口，不放宽权限
- forbidden、not found、error 状态明确可见

### 5.2 当前信息架构

当前结构：

- tenant / workspace
- knowledge base
- document tree
- 独立阅读页
- 独立编辑页

当前页面集合：

- 登录页
- 工作台首页
- 知识库详情页
- 文档阅读页
- 文档编辑页

### 5.3 当前核心功能

#### 工作台

- 当前租户工作台聚合
- 最近编辑文档入口
- 知识库列表入口
- 轻量概览信息
- 租户成员列表

#### 知识库

- 知识库列表与详情
- 创建 / 编辑 / 删除
- 知识库级权限覆盖
- 知识库成员配置

#### 文档

- 统一文档 / 文件夹模型
- 文档树浏览
- 创建 / 重命名 / 移动 / 删除
- 拖拽排序与批量整理
- 独立阅读页
- 独立编辑页
- 阅读链接复制

#### 版本

- 自动版本快照
- 版本列表
- 按版本回滚
- 基础 diff

#### 权限

- bearer token demo 会话占位
- 租户角色控制
- 知识库级权限覆盖
- 前端最小权限感知

### 5.4 当前成熟度判断

基础较稳：

- 多租户边界
- 多知识库模型
- 文档树管理
- 基础阅读与编辑
- 版本快照与回滚

可演示闭环：

- 工作台
- 知识库管理
- 知识库级权限管理
- 阅读链接

尚未生产化：

- 真实认证
- 完整权限模型
- 全局搜索
- 受控阅读 / 发布
- 审计

## 6. 目标产品设计

### 6.1 目标定位

目标不是“再做一个 Notion”。

更合理的目标是：

`具备企业边界、清晰知识库结构、受控阅读和版本治理的在线文档平台`

### 6.2 目标信息架构

建议目标结构：

- tenant / workspace
- knowledge base
- document tree
- reader surface
- editor surface
- publishing surface

### 6.3 目标能力域

#### A. 知识管理

- 多知识库
- 模板
- 标签 / 分类
- 归档 / 回收站
- 文档引用关系

#### B. 阅读与编辑

- 阅读页
- 编辑页
- 版本历史
- 对比与恢复
- 评论 / 批注能力预留

#### C. 权限与发布

- 租户级权限
- 知识库级权限
- 文档级例外权限
- 受控阅读入口
- 对外发布与撤销

#### D. 搜索与发现

- 工作区搜索
- 知识库内搜索
- 标题与正文搜索
- 结果聚合与跳转

---

## 7. 范围边界

### 7.1 当前范围内

- 工作台
- 知识库管理
- 文档树管理
- 阅读 / 编辑双页
- 版本快照与回滚
- 最小权限控制

### 7.2 暂不纳入当前基线

- AI 问答主入口
- 资源库主入口
- 技能市场
- 通用项目管理协作
- 实时多人协同编辑
- 独立桌面产品线

---

## 8. 成功标准

当前阶段至少要回答清楚：

- 用户能否稳定进入工作台并进入知识库
- 用户能否稳定管理文档树
- 用户能否稳定阅读、编辑并生成版本
- 用户能否明确知道谁能读、谁能改
- Web 端是否已经具备清晰、连续、易理解的主操作流

如果这些问题答不清楚，说明当前产品闭环还没有完成。
