# 在线文档系统主流程冒烟验收清单

## 1. 目的

本文档用于固定当前在线文档系统最核心的人工验收路径，避免每次只靠口头描述“应该能用”。

当前聚焦的验收主链路是：

`admin 登录 -> 工作台 -> 创建知识库 -> 创建文档 -> 编辑保存 -> 查看版本 -> 复制阅读链接 -> 退出登录`

---

## 2. 验收前提

### 2.1 后端

启动命令：

```bash
./start-backend.sh
```

当前已知状态：

- 后端主链路的最小集成测试已具备
- `start-backend.sh` 默认会通过共享脚本优先使用 `./gradlew`
- 使用 wrapper 时会在仓库内设置 `GRADLE_USER_HOME`，避免写入用户主目录
- 如果本地既没有可用的系统 Gradle，也没有已缓存的 wrapper 分发包，后端启动仍会受本机 Gradle 环境限制
- 关键测试文件：
  - [OnlineDocumentApiIntegrationTest.java](../../memora-server/memora-server-start/src/test/java/com/memora/OnlineDocumentApiIntegrationTest.java)

### 2.2 Web

启动命令：

```bash
cd memora-web-app
npm install
npm run dev
```

当前已知状态：

- `npm run lint` 已通过
- `npm run build` 已通过

### 2.3 演示账号

- `admin / 123456`

## 4. 人工冒烟步骤

### 4.1 登录

步骤：

1. 打开 `/login`
2. 输入 `admin / 123456`
3. 点击登录动作

期望结果：

- 成功进入工作台 `/`
- 顶部导航可见，且可以退出当前会话
- 未登录访问受保护页面时会被重定向到登录页

### 4.2 工作台查看

步骤：

1. 进入工作台首页
2. 查看最近文档、最近编辑项、知识库列表

期望结果：

- 工作台加载成功
- 第一屏优先展示最近编辑文档入口
- 能看到当前租户的知识库列表
- `admin` 可以点击“新建知识库”

### 4.3 创建知识库

步骤：

1. 在工作台点击新建知识库
2. 输入名称和描述
3. 提交表单

期望结果：

- 自动跳转到新知识库详情页
- 侧边栏知识库导航自动刷新
- 顶部在线文档入口不再硬编码 `/kb/1`，而是进入当前或最近知识库

### 4.4 创建文档

步骤：

1. 在知识库详情页点击新建文档
2. 只输入标题
3. 提交创建

期望结果：

- 新节点出现在文档树中
- 新文档自动选中
- 如果是文档节点，自动进入独立编辑页
- 摘要和父级目录不是创建前置条件

### 4.5 编辑保存

步骤：

1. 打开独立编辑页
2. 在富文本编辑器中修改正文
3. 点击保存并生成版本

期望结果：

- 出现保存成功提示
- 首屏主要区域聚焦正文编辑
- 页面不再把知识库级信息和侧栏导航作为主焦点
- 文档版本号递增
- 返回知识库后，摘要和正文预览刷新

### 4.6 查看版本

步骤：

1. 在独立编辑页点击查看版本
2. 选择一个历史版本进行对比

期望结果：

- 能看到新生成的版本记录
- 版本按时间倒序展示
- 当前版本与历史版本差异可见
- 编辑页可直接切换到阅读页，不需要先回知识库

### 4.7 复制阅读链接

步骤：

1. 在知识库正文区或独立编辑页点击复制阅读链接
2. 执行复制
3. 在已登录且具备权限的会话中打开链接

期望结果：

- 可复制独立文档阅读链接
- 复制阅读链接不需要先进入编辑模式
- 打开的目标是当前文档阅读页，不是知识库首页
- 接收方不需要先进入知识库再手动找文档
- 链接本身不会放宽权限，未登录或无权访问的用户仍会被拒绝

### 4.8 退出登录

步骤：

1. 点击顶部退出动作

期望结果：

- 返回登录页或使受保护页面失效
- 本地会话数据被清除
- 最近访问知识库记录被清除

---

## 5. 当前重点回归项

人工验收时重点确认以下风险没有回归：

- 登录后知识库导航是否仍有硬编码跳转
- 创建知识库后是否还需要回列表再手动进入详情
- 创建文档后是否还需要额外点一次编辑器
- 保存正文后版本列表是否刷新
- 受限角色是否正确禁用按钮并拒绝未授权操作

---

## 6. 当前结论

截至当前代码状态：

- 后端主链路已有自动化回归保障
- Web 端主链路具备人工冒烟基础
- 完整权限产品化、高级协作编辑仍未完成

当前正确的验收方式是：

`先反复走通这份清单，再考虑扩展外围能力`
