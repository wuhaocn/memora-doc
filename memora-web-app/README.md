# Memora Web App

`memora-web-app` 是 Memora 在线文档系统的 Web 端，当前定位不是完整协作办公前端，而是：

`继续写作入口 + 专属知识库工作区 + 文档阅读/编辑双模式`

---

## 当前职责

Web 端当前主要负责：

- 展示当前租户工作台与最近编辑文档
- 提供知识库创建、进入和基础管理入口
- 提供知识库内文档树浏览与维护
- 提供独立文档阅读页和独立文档编辑页
- 提供版本查看、回滚和租户内只读分享入口
- 提供知识库级权限和本地同步基础入口

当前还不承担：

- 实时协同编辑
- 完整文档编辑体验
- AI 主流程
- 资源库主流程

## 当前页面结构

```text
memora-web-app/
├── src/
│   ├── components/
│   │   └── Layout/                  # 全局布局、顶部导航、知识库侧栏
│   ├── pages/
│   │   ├── Home/                    # 工作台首页
│   │   ├── Document/                # 独立文档编辑页
│   │   ├── KnowledgeBase/           # 知识库详情
│   │   └── NotFound/                # 404 页面
│   ├── contexts/                    # 当前会话上下文
│   ├── router/                      # 主路由
│   ├── services/
│   │   ├── api/                     # 后端 API 封装
│   │   └── http/                    # Axios 实例
│   ├── styles/                      # 全局样式
│   └── utils/                       # demo 会话存储与辅助函数
```

当前主路由：

- `/login` 登录页
- `/` 工作台
- `/kb/:id` 知识库详情
- `/docs/:documentId` 独立文档阅读页
- `/docs/:documentId/edit` 独立文档编辑页

## 当前界面设计方向

最新阶段状态记录：

- [workspace-ui-status-2026-04-12.md](./docs/workspace-ui-status-2026-04-12.md)

本轮重构的界面方向是：

- 继续编辑优先
- 布局参考知识文档工作台
- 顶部采用轻量上下文条，不再使用大块运营型 hero
- 页面主体采用 左树 / 中央文档纸面 / 右辅助侧栏
- 中央正文统一使用白色 paper stage 承载阅读和编辑
- 整体配色采用蓝灰白工作区体系，主按钮和选中态统一使用蓝色
- 首页只保留高频入口
- 知识库管理次级呈现
- 同步和成员信息默认折叠

当前首页聚焦：

- 上次编辑文档直达入口
- 最近编辑文档
- 轻量概览条
- 知识库列表入口
- 默认折叠的同步信息
- 默认折叠的成员列表

当前知识库详情聚焦：

- 紧凑知识库上下文条，知识库说明按需展开
- 左侧文档树，支持目录展开 / 收起和轻量批量整理
- 中部正文阅读主舞台，标题和主动作共用一行
- 目录节点不再展示空白预览，而是直接给出“新建文档 / 新建目录”
- 查看版本改为按需展开侧栏
- 同步记录保持低频折叠展示
- 文档专注模式

当前文档编辑页聚焦：

- 只展示当前文档标题、保存状态和高频动作
- 不展示其他知识库信息和知识库侧栏
- 顶栏采用知识文档工作区同款轻顶栏
- 顶栏直接提供“返回知识库 / 阅读文档 / 分享文档 / 查看版本”
- 默认把页面空间优先让给正文编辑
- 内置租户内文档分享入口，可直接复制独立阅读链接

当前文档阅读页聚焦：

- 只保留阅读正文所需信息
- 常规阅读模式提供“返回知识库 / 分享文档 / 继续编辑”
- 分享阅读模式只保留只读访问，不展示编辑入口
- 分享后打开的是独立阅读页，不强制先回知识库查找

## 当前交互约束

页面定位：

- 首页是继续写作入口，不是知识库管理看板
- 知识库详情页是文档工作区，不是后台详情页
- 文档编辑页只服务编辑本身
- 分享阅读页只服务只读访问

按钮命名：

- 知识库入口统一使用“进入知识库”
- 知识库和文档创建统一使用“新建”
- 文档正文查看统一使用“阅读文档”
- 文档主动作统一使用“继续编辑 / 分享文档 / 查看版本”
- 回到知识库工作区统一使用“返回知识库”

低频信息：

- 版本记录、同步记录、成员和权限默认按需展开
- 只有主流程动作可以占据主按钮位
- 弹窗和抽屉延续页面主流程语言，不再使用后台型英文辅助标题

样式约定：

- 轻标签和轻状态标记优先复用 [global.css](./src/styles/global.css) 中的 `ui-chip` / `ui-chip-accent`
- 页面级 CSS Module 只补充间距、布局和局部差异，不再重复定义同一类轻标签样式
- 知识库详情、阅读页、编辑页统一使用浅灰工作区背景 + 白色文档纸面 + 细边框侧栏
- 主按钮、选中态、展开态统一使用蓝色高亮，不再混用暖色渐变

## 本地开发

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

默认依赖后端地址：

- `http://localhost:8080`

Axios 基础配置：

- [axios.js](./src/services/http/axios.js)

当前会默认通过 bearer token 传递 demo 会话：

- `Authorization: Bearer demo:{tenantId}:{userId}`

当前演示登录账号：

- `admin / 123456`

同时仍兼容旧的请求头上下文：

- `X-User-Id`
- `X-Tenant-Id`

前端启动后会调用：

- `POST /api/v1/auth/login`
- `GET /api/v1/auth/session`

用于登录系统并把本地会话补全为当前租户会话信息，再通过 `AuthContext` 分发给页面。

## 当前技术边界

Web 端当前是“管理台”，不是“最终协作终端”。

这意味着后续优先级应当是：

1. 在现有知识库管理基础上补齐校验、反馈和权限约束
2. 在现有基础编辑器上补齐高级编辑能力与协作体验
3. 把同步结果展示闭环做完整
4. 再补权限、搜索、发布和更细粒度的版本能力
